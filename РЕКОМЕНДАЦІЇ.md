# Рекомендації з розвитку проєкту «Мій розклад уроків»

## Що вже виправлено

- **Безпека (XSS):** назви уроків і посилання екрануються при виводі; посилання перевіряються (лише `http://` та `https://`).
- **Скидання:** після «Скинути все» очищуються вибір дня та список уроків.
- **Мета-теги:** додано `viewport` для коректного відображення на мобільних; прибрано зайвий коментар.
- **Надійність:** обробка пошкоджених даних у `localStorage` (try/catch при `JSON.parse`), перевірка `schedule[day]` та полів уроку.
- **Доступність:** кнопки мають `type="button"`; посилання відкриваються з `rel="noopener noreferrer"`.
- **Код:** використання `data-*` атрибутів замість рядків у `onclick`/`oninput` зменшує ризик помилок і ламаного коду.

## Рекомендації для подальшого розвитку

### 1. Структура проєкту
- Винести CSS у окремий файл `styles.css` і підключити через `<link>`.
- Винести JavaScript у `script.js` і підключити перед `</body>` (або з `defer`).
- Це спростить підтримку, кешування і можливе використання мініфікаторів.

### 2. Доступність (a11y)
- Додати `<label for="...">` для `select` або обгорнути його в `<label>`.
- Для кнопки видалення — `aria-label="Видалити урок"` (або аналог).
- Для блоків уроків — семантичні теги: `<article>`, `<section>` або `<fieldset>` замість лише `<div>`.

### 3. UX
- Підтвердження перед видаленням одного уроку («Видалити цей урок?»).
- Можливість змінювати порядок уроків (перетягування або кнопки «вгору»/«вниз»).
- Поле часу уроку (наприклад, початок і кінець).
- Експорт/імпорт розкладу (JSON або файл), щоб не втратити дані при зміні браузера/пристрою.

### 4. Стилі та адаптивність
- Додати медіа-запити для екранів &lt; 480px (ширина блоків, розміри шрифтів, відступи).
- Використати `max-width` замість фіксованої `width: 420px` для блоків уроків.
- Можливість теми (світла/темна) через CSS-змінні або перемикач.

### 5. JavaScript
- Обгорнути код у IIFE або модуль, щоб не засмічувати глобальну область.
- Додати перевірку на наявність `schedule[day]` при оновленні/видаленні (захист від невідповідності структури даних).
- За потреби — дебаунс для `oninput`, щоб рідше писати в `localStorage` при швидкому введенні.

### 6. Додаткові функції
- Фільтр/пошук по назві уроку в межах тижня.
- Підсвітка поточного дня.
- Нагадування (з використанням Notification API або інтеграція з календарем).
- Підтримка кольорів/іконок для предметів.

### 7. Якість коду
- Додати коментарі для ключових функцій (укр. або англ.).
- Розглянути TypeScript або JSDoc для типізації об’єктів розкладу та уроків.
