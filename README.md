# Мій розклад уроків

Веб-додаток для ведення особистого розкладу уроків у браузері. Підтримує кілька днів тижня, посилання на онлайн-уроки (Zoom, Meet тощо), час початку та закінчення, кольорові позначки та експорт/імпорт даних.

## Що це за проєкт

- **Мій розклад уроків** — статичний односторінковий додаток (HTML + CSS + JavaScript).
- Дані зберігаються лише у вашому браузері (localStorage), нікуди не відправляються.
- Працює офлайн після першого відкриття сторінки.

## Як запустити

1. Клонуйте репозиторій або завантажте файли проєкту.
2. Відкрийте `index.html` у браузері (подвійний клік або через локальний сервер).

Локальний сервер (не обов’язково, але зручно для розробки):

```bash
# Python 3
python -m http.server 8000

# Node (npx)
npx serve .
```

Потім відкрийте в браузері: `http://localhost:8000`.

## Структура проєкту

```
schedule-app/
  index.html   # Розмітка та підключення стилів і скриптів
  styles.css   # Усі стилі (включно з темою та адаптивністю)
  script.js    # Логіка додатку (збереження, відображення, експорт/імпорт)
  README.md    # Ця інструкція
```

## Як зберігаються дані

- Дані зберігаються в **localStorage** під ключем `scheduleData`.
- Формат — JSON: об’єкт з ключами днів тижня (Понеділок, Вівторок, …), кожен ключ — масив уроків.
- Кожен урок містить: назву, посилання, час початку, час закінчення, колір.
- При пошкодженні даних додаток відновлює порожній розклад і перезаписує localStorage.

### Синхронізація з будь-якого пристрою (GitHub Gist)

Щоб відкривати той самий розклад на телефоні, планшеті або іншому комп’ютері:

1. Розгорніть блок **«Синхронізація з будь-якого пристрою (GitHub Gist)»** на сторінці.
2. Перейдіть за посиланням «Створити токен на GitHub» і створіть **Personal Access Token** з правами **gist** (класичний токен або fine-grained).
3. Скопіюйте токен і вставте в поле на сторінці, натисніть **Підключити**.
4. Після цього розклад зберігається в вашому **приватному Gist** на GitHub і автоматично синхронізується при змінах.
5. На іншому пристрої можна підключитися двома способами:
   - **QR-код:** на пристрої, де вже підключено синхронізацію, розгорніть блок і покажеться QR-код. На іншому пристрої натисніть **«Сканувати QR-код»**, наведіть камеру на екран із QR або завантажте фото — токен підставиться автоматично.
   - **Вручну:** вставте той самий токен у поле й натисніть **Підключити**.

Токен зберігається лише у вашому браузері (localStorage) і використовується лише для доступу до вашого Gist. Нічого окремого реєструвати не потрібно — достатньо вашого облікового запису GitHub.

## Експорт та імпорт

- **Експорт:** кнопка «📤 Експорт» зберігає поточний розклад у JSON-файл на ваш комп’ютер.
- **Імпорт:** кнопка «📥 Імпорт» відкриває вибір файлу; після вибору валідного JSON-файлу розклад замінюється імпортованими даними.

Це зручно для резервних копій або перенесення розкладу на інший пристрій/браузер.

## Як розгорнути (GitHub Pages, Netlify, Cloudflare Pages)

Додаток повністю статичний — достатньо завантажити вміст папки `schedule-app/` на хостинг.

### GitHub Pages

1. Створіть репозиторій, завантажте файли з `schedule-app/` (або зберігайте їх у корені або в папці `docs/`).
2. У репозиторії: **Settings → Pages**.
3. **Source:** Deploy from a branch. Виберіть гілку (наприклад `main`) та папку з файлами (наприклад `/ (root)` або `docs`).
4. Збережіть — через хвилину сайт буде доступний за посиланням виду `https://<username>.github.io/<repo>/`.

### Netlify

1. Зареєструйтесь на [netlify.com](https://netlify.com).
2. **Add new site → Deploy manually** або підключіть Git-репозиторій.
3. Вкажіть теку з проєктом (наприклад `schedule-app` або корінь, якщо там лежать `index.html`, `styles.css`, `script.js`).
4. Збережіть — Netlify видасть посилання на сайт.

### Cloudflare Pages

1. У [Cloudflare Dashboard](https://dash.cloudflare.com) виберіть **Pages → Create a project**.
2. Підключіть репозиторій або завантажте папку (Upload assets).
3. Build settings: якщо це лише статичні файли — Build command можна залишити порожнім, Output directory — корінь проєкту.
4. Після деплою отримаєте URL виду `https://<name>.pages.dev`.
